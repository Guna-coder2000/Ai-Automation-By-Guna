TARGET REQUIREMENT MODES

The target requirement determines locator uniqueness expectations.

Mode 1: Single Element (1 of 1)

* Locator must identify exactly one element.
* Return the most stable unique strict XPath (e.g., //input[@placeholder='...']).
* Do NOT return CSS selectors. You MUST return an XPath.
* Validate uniqueness before returning.

Examples:

* Login button
* Username input
* Submit button

Mode 2: Element Collection (1 of All)

* Locator must identify all matching elements of the requested type.
* Do not force uniqueness.
* Return the most stable common locator pattern.
* Locator should be reusable for collections, lists, tables, menus, cards, rows, or repeated components.

Examples:

* All product cards
* All table rows
* All menu items
* All checkboxes
* All buttons inside a section

TARGET IDENTIFICATION PROCESS

Before selecting a locator:

1. Identify the intended target element.
2. Determine why the original locator failed.
3. Analyze nearby elements.
4. Compare attributes.
5. Detect UI framework patterns.
6. Detect dynamic attributes.
7. Detect generated IDs.
8. Detect accessibility metadata.
9. Detect dynamic CSS selectors.
10. Detect dynamic XPath patterns.
11. Determine whether the requirement is:

    * Single Element (1 of 1)
    * Element Collection (1 of All)
12. Validate locator suitability.
